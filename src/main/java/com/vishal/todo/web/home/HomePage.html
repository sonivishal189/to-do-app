This application is a To-Do application that is designed to track the tasks of a construction company called "Dream House Realty". It has been developed using Java, Apache Wicket, Hibernate, Maven, Jetty, Bootstrap, HTML, and PostgresDB.
The application offers various features, including a login page with basic authentication where users can enter their email and password to access the application. It also includes a registration feature for new users. If the entered credentials are incorrect, an appropriate error message is displayed on the screen. Upon successful login, users are redirected to the Home page.
The homepage features a "Go to To-Do Application" button that leads to the main to-do application. The main application exhibits all the tasks that are currently available, categorized by their respective status. Additionally, it provides a count of all tasks and tasks in each status category. On this page, there are buttons available to perform various operations such as creating, viewing, editing, and deleting tasks.

Features in To-Do Application

Create Task

  1. User can create a task by clicking on the "Create Task" button.
  2. Clicking the button opens a prompt that requires three fields: Task Title, Assignee Name, and Building Name. Task title and building name are mandatory fields.
  3. The user needs to select the Assignee Name and Building Name from the dropdown options.
  4. After filling in the details, the user should click on the "Create New Task" button in the prompt to save the task. This also update the task counts on the screen.
  5. If the Assignee Name is not provided during task creation, the task status is considered "New"; otherwise, it is considered "Assigned".
  6. The application also saves the details of the user who created the task, along with the task creation timestamp.
  7. The creator name is taken from the loggedIn user's details.

View Task

  1. The main application page allows users to view tasks by selecting 1 or more tasks and clicking "View Task" button.
  2. Users need to select the desired task by clicking on its corresponding checkbox.
  3. View task page displays all the information about the selected tasks. This includes the task id, task title, assigned person, building name associated with the task, task status, task creator, creation date, task updater, and update date.
  4. User must select at least one task before clicking the 'View Task' button, if not then an error message will be displayed.
  5. View task page provides "Back" button to navigate back to main application page.

Edit Task

  1. To edit a task, the user needs to select the task by clicking on the corresponding checkbox. and click on the "Edit Task" button.
  2. Clicking on the edit button redirects the user to the edit task page which populates the details of the selected task.
  3. The user can edit the task title, status, assignee name, and building name. The assignee name, status, and building name is selected from dropdown options.
  4. After editing, the user can submit the changes and applciation updates the task details in database and also udpate the correponding task count on the screen.
  5. The application also updates the details of the user who updated the task, along with the task updation timestamp.
  6. The updater name is taken from the loggedIn user's details.
  7. User can only edit 1 task at a time.

Delete Task

  1. The user has the ability to delete one or multiple tasks at once by selecting the corresponding task checkboxes and then clicking on the "Delete Task" button.
  2. It is essential for at least one task to be selected in order to initiate the delete operation successfully.
  3. Once the user clicks on the delete button, the task record associated with the selected task(s) will be removed from the database.
  4. Additionally, the task count displayed on the interface will be promptly updated to reflect the modified number of remaining tasks following the deletion process.
